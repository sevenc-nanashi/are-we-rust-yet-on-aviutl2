## Design Context

### Users

People casually checking how much of the AviUtl2 plugin and script catalog is Rust-powered. They are browsing, not operating a complex dashboard.

### Brand Personality

Plain, curious, and catalog-like. The page should feel close to the classic "Are we X yet?" Rust ecosystem pages: direct, lightweight, and slightly old-web.

### Aesthetic Direction

Use a minimal light page with strong text hierarchy, a single percentage bar, and a simple repository list. Avoid marketing sections, dashboard cards, decorative imagery, and unnecessary metadata.

### Design Principles

- Keep the first screen focused on the question and the answer.
- Prefer readable document structure over app-like chrome.
- Use one strong Rust-colored accent only where it carries meaning.
- Show repositories as a browsable list, not as cards.
- Do not add explanatory copy unless it is needed for state or accessibility.
